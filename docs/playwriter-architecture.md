# Playwriter の仕組み -- やさしい解説

## この文書について

本アプリ（web-automation-3）がブラウザを操作するために使っている「Playwriter」という仕組みを、できるだけ平易な言葉で説明します。

## 登場人物は3人

本アプリがブラウザを操作するには、3つのプログラムが協力して動きます。

### 1. Chrome拡張（Playwriter拡張）

Chromeブラウザにインストールする小さなプログラムです。ブラウザの中で動いていて、「ページのボタンをクリックしろ」「今のページの情報を教えろ」といった命令を受け取って実行できます。

ただし、Chrome拡張は外の世界と自由に通信できません。そこで次の「リレーサーバー」が必要になります。

### 2. リレーサーバー（playwriter serve）

ターミナルで `npx playwriter serve --host 127.0.0.1` と打って起動する、中継役のプログラムです。ポート19988番で待機し、Chrome拡張と本アプリの間でメッセージを橋渡しします。

イメージとしては「通訳さん」です。Chrome拡張の言葉と、本アプリの言葉を翻訳して伝えてくれます。

### 3. 本アプリ（web-automation-3）

`pnpm wa status` などのコマンドを実行するプログラムです。「ブラウザの今のページURLを教えて」といった命令を出す側です。

## どうやって繋がるのか

3人がどう繋がっているか、図で示します。

```
本アプリ（pnpm wa status）
  |
  | 本アプリが playwriter --host 127.0.0.1 を
  | 裏で自動的に起動する（ユーザーは意識しない）
  |
  v
MCPサーバー（playwriter --host 127.0.0.1）
  |
  | インターネット通信（WebSocket）で
  | リレーサーバーに接続する
  |
  v
リレーサーバー（playwriter serve、ポート19988）
  |
  | Chrome拡張もここに繋がっている
  |
  v
Chrome拡張（ブラウザ内で動作）
  |
  | ブラウザを実際に操作する
  |
  v
ブラウザのページ（Google、社内サイトなど）
```

## 通信の仕組み：MCP と stdio

### MCP（Model Context Protocol）とは

AIやプログラムが外部のツール（ブラウザ、ファイル、データベースなど）を操作するための共通ルール（通信規格）です。「こういう形式でお願いを送って、こういう形式で返事をもらう」という約束事です。

### stdio とは

「standard input / output」の略で、日本語では「標準入出力」です。プログラム同士がメッセージをやり取りする最もシンプルな方法です。

普段ターミナルでキーボードから文字を打ち込むのが「標準入力（stdin）」、画面に文字が表示されるのが「標準出力（stdout）」です。プログラム同士でも、一方のプログラムが相手のstdinに文字を送り、相手のstdoutから返事を読むことで会話できます。

本アプリでは、MCPのメッセージをstdioでやり取りしています。

## 実際の流れ：`pnpm wa status` を実行すると

1. 本アプリが `playwriter --host 127.0.0.1` を裏で起動する
2. 本アプリが「ページのURLとタイトルを教えて」という命令を作る
3. この命令をJSON形式にして、playwriterプロセスのstdinに送る
4. playwriterプロセスがリレーサーバー（ポート19988）に転送する
5. リレーサーバーがChrome拡張に転送する
6. Chrome拡張がブラウザのページ情報を取得する
7. 結果が逆の経路で戻ってくる（Chrome拡張 → リレー → playwriter → 本アプリ）
8. 本アプリがターミナルに結果を表示する

## ユーザーが事前にやること

本アプリを使う前に、以下の準備が必要です。

### 一度だけやること

- ChromeにPlaywriter拡張をインストールする

### 毎回やること

1. Chromeで操作したいページを開く
2. Playwriter拡張のアイコンをクリックして有効化する（アイコンが緑色になればOK）

Playwriterに接続するコマンド（`wa status`、`wa record` など）を実行すると、リレーサーバーは自動起動されます。手動で `npx playwriter serve` を実行する必要はありません。

Chrome拡張のアイコンをクリックするタイミングは、コマンド実行の前後どちらでも構いません。拡張にはリトライロジックがあり、サーバーが起動すれば自動的に接続されます。

## よくある質問

### リレーサーバーは必ず必要？

はい。Chrome拡張はセキュリティ上の理由で外部プログラムと直接通信できないため、リレーサーバーが中継役として必要です。

### playwriter拡張のアイコンが緑にならない

リレーサーバーが起動していない可能性があります。`pnpm wa status` を実行すると、リレーサーバーが自動起動されます。その後、拡張アイコンをクリックしてください。

### MCP serverはAI専用？

いいえ。MCPはもともとAI Agentが外部ツールを操作するために設計されましたが、通信規格はただのJSON-RPCなので、本アプリのように普通のプログラムからも使えます。

## 用語まとめ

| 用語 | 意味 |
|------|------|
| Playwriter | ブラウザ操作ツール。Chrome拡張・MCP サーバー・リレーサーバーの3つの機能を提供するnpmパッケージ |
| Chrome拡張 | ブラウザ内で動く小さなプログラム。実際にページを操作する |
| リレーサーバー | Chrome拡張と外部プログラムの間でメッセージを中継するサーバー |
| MCP | プログラムが外部ツールを操作するための通信規格 |
| stdio | プログラム同士がstdin/stdoutでメッセージをやり取りする方法 |
| CDP | Chrome DevTools Protocol。Chromeを遠隔操作するためのプロトコル |
| JSON-RPC | JSONでリクエストとレスポンスをやり取りするシンプルな通信方式 |
